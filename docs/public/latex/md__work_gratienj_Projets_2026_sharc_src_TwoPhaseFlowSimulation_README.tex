Le module {\ttfamily Two\+Phase\+Flow\+Simulation} implémente le solveur principal de Sh\+Arc pour la {\bfseries{simulation d’écoulements diphasiques en milieu poreux}} (pression + saturations), ainsi qu’un module optionnel de {\bfseries{convection thermo‑chimique couplée}} (pression, concentration, température).

Il s’appuie sur le modèle de données {\bfseries{Arc\+Res}} pour décrire les systèmes physiques (fluides et solide), et sur le framework {\bfseries{Arc\+Geo\+Sim/\+Arcane}} pour la discrétisation numérique (schéma deux‑points, transmissivités, opérateurs de type {\ttfamily Div\+KGrad}) et la résolution non linéaire ({\ttfamily INewton\+Solver}). La structure du module est entièrement pilotée par les fichiers {\ttfamily .axl} (descripteurs \mbox{\hyperlink{namespaceArcane}{Arcane}}) qui enregistrent les points d’entrée ({\ttfamily init}, {\ttfamily compute}) et la liste des services (conditions initiales, lois physiques, conditions aux limites, puits, etc.).

Le module organise la simulation autour d’un système non linéaire {\ttfamily \mbox{\hyperlink{classArcNum_1_1INonLinearSystem}{Arc\+Num\+::\+INon\+Linear\+System}}} \+:
\begin{DoxyItemize}
\item construction de la matrice et du résidu à chaque itération Newton (accumulation, flux internes, flux de bord, flux de puits, équation de fermeture),
\item gestion des variables d’état dans des \char`\"{}folders\char`\"{} de cellules/faces ({\ttfamily Law\+::\+Variable\+Cell\+Folder}, {\ttfamily Partial\+Variable\+Face\+Folder}),
\item gestion de la {\bfseries{restauration de l’état}} en cas d’échec du solveur (snapshots temporels, {\ttfamily Replay\+Exception}).
\end{DoxyItemize}

Le répertoire encapsule également la logique de {\bfseries{conditions aux limites}}, {\bfseries{conditions initiales}}, {\bfseries{puits}} et {\bfseries{lois physiques}} (pression capillaire, densité fluide, perméabilité relative, loi constante), ainsi que des interfaces génériques pour la construction du système physique ({\ttfamily \mbox{\hyperlink{classIPhysicalSystem}{IPhysical\+System}}}, {\ttfamily \mbox{\hyperlink{classISystemBuilder}{ISystem\+Builder}}}, {\ttfamily \mbox{\hyperlink{classIVariableManager}{IVariable\+Manager}}}).\hypertarget{md__work_gratienj_Projets_2026_sharc_src_TwoPhaseFlowSimulation_README_autotoc_md155}{}\doxysection{Fonctionnalités}\label{md__work_gratienj_Projets_2026_sharc_src_TwoPhaseFlowSimulation_README_autotoc_md155}
Basé sur les symboles et services détectés \+:


\begin{DoxyItemize}
\item ✅ Simulation diphasique principale via {\ttfamily Two\+Phase\+Flow\+Simulation\+Module} \+:
\begin{DoxyItemize}
\item Construction d’un système non linéaire sur pression + saturations.
\item Calcul de l’accumulation, des flux internes, des flux de bord et des flux de puits.
\item Équation de fermeture (somme des saturations = 1).
\item \mbox{\hyperlink{namespaceApplication}{Application}} de contraintes physiques sur la solution (projection des saturations dans \mbox{[}0,1\mbox{]}, renormalisation).
\end{DoxyItemize}
\item ✅ Simulation thermo‑chimique couplée via {\ttfamily Thermo\+Chemical\+Convection\+Module} \+:
\begin{DoxyItemize}
\item Équations pour pression, concentration et température.
\item Accumulation couplée fluide/solide (chaleur volumique, capacité thermique).
\item Flux d’advection (Darcy) et de diffusion moléculaire pour la concentration.
\item Flux de chaleur conductif (fluide/solide) en plus de l’advection.
\end{DoxyItemize}
\item ✅ Gestion des {\bfseries{conditions initiales}} \+:
\begin{DoxyItemize}
\item Services {\ttfamily \mbox{\hyperlink{classIInitialCondition}{IInitial\+Condition}}} appliqués sur les variables de domaine (pression, saturations, concentration, température, etc.).

\begin{DoxyItemize}
\item Interface {\ttfamily \mbox{\hyperlink{classIBoundaryManager}{IBoundary\+Manager}}} et service {\ttfamily Dirichlet\+Manager\+Service}.
\item Dossiers {\ttfamily Boundary\+Condition/$\ast$} pour les différents types de bord (Dirichlet, etc.).
\item Évaluation des lois physiques sur les faces de bord (densité, viscosité, perméabilité relative, pression capillaire).
\end{DoxyItemize}
\item ✅ Gestion des {\bfseries{puits}} ({\ttfamily Well\+Condition}) \+:
\begin{DoxyItemize}
\item Interface {\ttfamily \mbox{\hyperlink{classIWellManager}{IWell\+Manager}}}, services de gestion des puits (débits, index de puits via {\ttfamily \mbox{\hyperlink{classWellIndexComputer}{Well\+Index\+Computer}}}).
\item Flux puits (terme source/sink) intégrés dans le résidu.
\end{DoxyItemize}
\item ✅ Modèle de {\bfseries{système physique Arc\+Res}} \+:
\begin{DoxyItemize}
\item Interfaces {\ttfamily \mbox{\hyperlink{classIPhysicalSystem}{IPhysical\+System}}}, {\ttfamily \mbox{\hyperlink{classISystemBuilder}{ISystem\+Builder}}}, {\ttfamily \mbox{\hyperlink{classIVariableManager}{IVariable\+Manager}}}.
\item Dossier {\ttfamily Physical\+System/} pour le couplage Arc\+Res ↔ simulation (domaines, phases, variables).
\item {\ttfamily User\+System\+Service} pour brancher un système utilisateur configuré.
\end{DoxyItemize}

\item Configurateurs {\ttfamily \mbox{\hyperlink{classILawConfigurator}{ILaw\+Configurator}}} et services {\ttfamily $\ast$\+Config\+Service}.

\item Mise à jour explicite de la géométrie ({\ttfamily \mbox{\hyperlink{classManualUpdateGeometryPolicy}{Manual\+Update\+Geometry\+Policy}}}), notamment après la création de groupes.
\end{DoxyItemize}
