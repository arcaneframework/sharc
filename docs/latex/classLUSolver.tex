\doxysubsection{Detailed Description}
\subsubsection*{template$<$typename T$>$\newline
class LUSolver$<$ T $>$}

Solve the linear system $A x = b$ using the $LU$ factorization. 

\begin{DoxyAuthor}{Author}
Daniele A. Di Pietro 
\end{DoxyAuthor}
\begin{DoxyDate}{Date}
2006-\/07-\/29
\end{DoxyDate}
The dense $LU$ solver can be used in one of the following ways\+:


\begin{DoxyItemize}
\item if the matrix $A$ needn\textquotesingle{}t be preserved after computation and the associated system has to be solved only once, simply call solve(\+A, b). At exit, the argument A will contain the $LU$ factorization of the original matrix. \mbox{\hyperlink{structIn}{In}} order to avoid vector/matrix copies, the \mbox{\hyperlink{classLUSolver_a80014669ce4f3a7d619482fa16b8726c}{overwrite\+\_\+solve()}} members may be used instead, the sole difference being that the solution is overwritten over the rhs vector/matrix. When the matrix $A$ passed as an argument is stored column-\/wise, the $L$ and $U$ factors are overwritten in the lower and upper triangular part of $A$ respectively. When $A$ is stored row-\/wise, the $U^t$ and $L^t$ factors are stored in the lower and upper triangular part of $A$ respectively.
\item if the matrix $A$ needs to be preserved after the computation or the associated system has to be solved several times, consider using a two-\/step strategy by first invoking factor(\+A), which makes a local copy of the matrix and factor it, and the calling solve(b) member, which actually solves the associated system. At exit, the matrix A will be preserved and its factorization will be stored in the \mbox{\hyperlink{classLUSolver}{LUSolver}} object. When \mbox{\hyperlink{classLUSolver_a7741643a6011103164d8d4e3de66d3ab}{solve()}} is replaced by \mbox{\hyperlink{classLUSolver_a80014669ce4f3a7d619482fa16b8726c}{overwrite\+\_\+solve()}}, the solution is overwritten on the provided argument. 
\end{DoxyItemize}
