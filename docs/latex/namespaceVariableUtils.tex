\doxysubsection{Detailed Description}
Utilitaire de manipulation des variables arcane. 

Classe \mbox{\hyperlink{classVariableUtils_1_1VariableView}{Variable\+View}} \+: pour obtenir une vue sur une variable et la manipuler sans en connaitre le type (port{\ucr}e par le maillage ou non, partielle, scalaire, vectorielle...) et sans connaitre le type de donnees.

Permet de manipuler les variables arcane port{\ucr}es ou non par le maillage (copies, extraction d\textquotesingle{}un tableau ..). Les m{\ucr}thodes retournent des IVariable {\ucr} partir desquelles on peut cr{\ucr}er des r{\ucr}f{\ucr}rences (\mbox{\hyperlink{classVariableRef}{Variable\+Ref}}). Si la r{\ucr}f{\ucr}rence est construite avec un Variable\+Build\+Info sans pr{\ucr}ciser de propri{\ucr}t{\ucr}, les propri{\ucr}t{\ucr}s seront perdues et en particulier la variable sera d{\ucr}truite avec la r{\ucr}f{\ucr}rence (perte de la propri{\ucr}t{\ucr} IVariable\+::\+PPersistant). Il est donc conseill{\ucr} de construire la r{\ucr}f{\ucr}rence directement {\ucr} partir de l\textquotesingle{}IVariable ou en utilisant ivariable-\/$>$property en entr{\ucr}e dans le Variable\+Build\+Info, ou alors d\textquotesingle{}utiliser la fonction create\+Variable\+Ref et de stocker la r{\ucr}f{\ucr}rence. Dans le cas des m{\ucr}thodes ou l\textquotesingle{}on retrourne la r{\ucr}f{\ucr}rence, les variables ne sont pas construites avec la propri{\ucr}t{\ucr} IVariable\+::\+PPersistant. Dans ce cas l{\ucr}, Les variables seront d{\ucr}truites quand un delete sera effectu{\ucr} sur le pointeur. Pour d{\ucr}truire la variable on peut annuler la propri{\ucr}t{\ucr} IVariable\+::\+Persistant {\ucr} partir d\textquotesingle{}une \mbox{\hyperlink{classVariableRef}{Variable\+Ref}} \+: \mbox{\hyperlink{classVariableRef}{Variable\+Ref}} ref(ivariable); ref.\+unset\+Property(\+IVariable\+::\+Persistant); La variable sera alors d{\ucr}truit avec la r{\ucr}f{\ucr}rence.
